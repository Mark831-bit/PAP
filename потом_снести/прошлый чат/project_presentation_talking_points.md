Копилка технических деталей которые можно раскрыть на защите PAP.

**Индексы в таблице `presencas`:**
- `INDEX idx_data (data)` — ускоряет `GROUP BY data` для графика "посещений по дням". Без индекса — full table scan, с индексом — почти мгновенно.
- `INDEX idx_login (login)` — ускоряет `WHERE login = ?` для показа истории конкретного ученика/учителя.
- Рассказать: "я намеренно добавил индексы по тем колонкам по которым идёт фильтрация и группировка".

**Денормализация `nome` в `presencas`:**
- Храню `nome` копией прямо в строке вместо JOIN с `alunos`/`professores`.
- Плюс: быстрее вывод + история сохраняется если ученика удалили.
- Минус: если переименовать человека — старые записи останутся со старым именем (это нормально для истории).

**Why:** Пользователь явно попросил запоминать такие мелочи — проект небольшой и на защите нужно будет "лить воду" по технической части.

**How to apply:** Когда принимаем решение которое улучшает производительность/безопасность/архитектуру — дописывать сюда короткий пункт + объяснение. На защите пользователь сможет достать эти факты.
